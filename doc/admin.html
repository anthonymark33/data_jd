jdaccess

 set DAN (db access name), user/pswd, and server for jd ops server intask - op runs in current J task server localhost:x - op runs in J task at port x SSH tunnel connects localhost:x to J task on server jdaccess 'test me/xx intask' NB. DAN test, user/pswd u/p jdaccess 'foo' NB. same as 'foo u/p intask' jdaccess 'zz me/xx localhost:65002' jdaccess '' NB. report jdaccess 0 NB. clear access

jdadmin

 make db folder available with default jdaccess jdadmin 'ab' NB. make ~temp/jd/ab available jdadmin '.../ab' NB. make .../ab available jdadmin '' NB. report jdadmin 0 NB. clear all admin and release all locks default jdaccess is for: ab u/p intask DAN (db access name) maps to folder,user/pswds,ops different DANs can access the same or different folders DAN rules set by admin.ijs in db folder creates default admin.ijs if it does not exist lock on db folder prevents interference by other tasks jdadmin '...' NB. gets lock jdadmin 0 NB. releases all locks

jdadminx

 delete old folder, create new db folder, do jdadmin jddeletefolderok allows delete that is not in ~temp/jd path jdadminx'ab' NB. create ~temp/jd/ab jddeletefolderok_jd_ '...' jdadminx'.../ab' NB. create .../ab

datatune

 insert/update calls datatune_tab in the db locale before doing the operation datatune_tab defined in custom.ijs arg is list of col names and names and values from the user result is col values for insert/update datatune_tab can be used in many ways: validate data - assert v>0 modify data - convert float dollars to cents derive new cols - y m d cols from datetime col tutorial datatune_tut

jdshare

 jdshare_jd_'rwrwrw' NB. rw or r- or -- for owner/group/others normally the Jd task account owns the db folder giving another account access to the db folder allows Jd tasks running as that account to access the db lock prevents problems with multiple Jd tasks running as the owner or another account from using the db at the same time adjust permissions to access the db from another account best done with host tools by someone who understands security jdshare might be useful in simple situations
