delete

delete table where

where - rows to delete:

rows
rows - jd'read jdindex from table where ...'
rows - jd'key /in table';pairs - key /in gets all matches 
NB. example delete
jdadminx'example'[olddan=:jdaccess''
jd'gen test f 10'
jd'delete f';3 5
jd'delete f';'x<3'
jd'delete f';'int';104 107
jdaccess olddan

update

update table where pairs

where - rows to update:

rows - _ indicates all rows
rows - jd'read jdindex from table where ...'
rows - jd'key table';pairs - key gets last matches

Warning: jd'read jdindex ...' returns rows in sort order and this may well not be the order of pairs. In general this form of update should be used only where 1 row is updated or where several rows are all updated with the same values.

NB. example update
jdadminx'example'[olddan=:jdaccess''
jd'gen test f 10'
jd'update f';6 7;'int';200 300
jd'update f';_;'float';7.7
jd'update f';'boolean=1';'byte';'x'
jd'update f';'int';'int';108 101 104;'byte';'cba'
'i b'=: {:"1 jd'read jdindex,boolean from f where int < 104'
jd'update f';i;'boolean';-.b NB. modify with indexes
jdaccess olddan

upsert

upsert table key pairs

key is 1 or more cols used to determine if a data row is used to update or insert.

NB. example upsert
jdadminx'example'[olddan=:jdaccess''
jd'createtable f'
jd'createcol f a int';i.5
jd'createcol f b int';23+i.5
jd'upsert f';'a';'a';2 5 6;'b';66 77 88 NB. 1 update and 2 inserts
jdaccess olddan

sort

sort [/desc] table col(s)

table columns are sorted in place by cols(s)

NB. example sort
jdadminx'example'[olddan=:jdaccess''
jd'gen test f 6'
jd'sort /desc f int'
jd'reads from f'
jd'sort /desc f boolean int'
jd'reads from f'
jd'sort f int'
jd'reads from f'
jdaccess olddan
